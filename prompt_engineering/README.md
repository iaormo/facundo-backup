## Core Principles

### The Formula
**Quality Output = Clear Instructions + Context + Examples + Constraints**

### The 5 Principles

1. **CLARITY** - Be specific about what you want
2. **CONTEXT** - Provide relevant background
3. **CONSTRAINTS** - Set boundaries and specifications
4. **EXAMPLES** - Show what good looks like (few-shot)
5. **ITERATION** - Treat prompting as a conversation

### Perfect Prompt Anatomy
```
[ROLE]
You are [expert role].

[CONTEXT]
[Background information and situation]

[TASK]
[Specific action to perform]

[FORMAT]
[Desired output structure]

[CONSTRAINTS]
- [Limitation 1]
- [Limitation 2]

[EXAMPLES]
[Sample input/output pairs]

[EVALUATION CRITERIA]
[What makes a good response]
```

## Advanced Techniques

### 1. Chain of Thought (CoT)
Prompt the model to show reasoning step by step.
- **When:** Complex reasoning, math, logic
- **Benefit:** More accurate, easier to debug

### 2. Few-Shot Prompting
Provide examples of desired input/output.
- **When:** Specific formatting, style matching
- **Benefit:** Consistent output

### 3. Zero-Shot with Definition
Define task clearly without examples.
- **When:** Simple tasks, avoiding bias
- **Benefit:** Faster, no example prep

### 4. Role Prompting
Assign a specific persona.
- **When:** Expert knowledge, specific tone
- **Benefit:** Expert perspective

### 5. Step-by-Step Decomposition
Break complex tasks into steps.
- **When:** Multi-step processes
- **Benefit:** Structured output

### 6. Self-Consistency
Generate multiple answers, select best.
- **When:** Critical decisions, fact-checking
- **Benefit:** Higher accuracy

### 7. Tree of Thoughts
Explore multiple reasoning paths.
- **When:** Strategic decisions
- **Benefit:** Risk assessment

## Optimization

### Iterative Refinement Process:
1. **Baseline** - Write initial prompt
2. **Evaluate** - Check output quality
3. **Refine** - Add specificity
4. **Test** - Run refined version
5. **Document** - Save successful prompts

### Common Issues & Fixes:
- **Too long** → Add word count constraint
- **Wrong format** → Specify structure
- **Too generic** → Add specificity requirements
- **Wrong tone** → Define tone explicitly
- **Hallucinations** → Add knowledge cutoff disclaimer

### A/B Testing:
Test variations to find optimal:
- Direct vs. Role-based
- With vs. Without examples
- Different framing approaches
